## 升级系统所遇到的问题

刷上类原生后发现手机就是流畅，朋友和我同一个型号的手机到2026年已经相对来说比较卡顿了（可能是云控或者后台占用大）。但是类原生有个很大的问题，就是升级系统的时候如果兼容旧版本系统的Data分区倒还好，如果不兼容就要清除Data分区内所有的东西，甚至要三清（Dalvik/ART 缓存，Data，Cache），而清除后就跟格式化一样手机内一切用户数据全部消失。就当你接受了格式化手机之后你会发现天塌了，由于防火长城的存在我们无法使用Google服务，但开机向导是要连接Google服务，否则无法完成开机向导。

## 我的一些方法
由于无法连接Google服务，现在的思路就是如何让它连接上Google服务或者在离线模式下完成向导。

一下有这几种方法可以尝试，虽然不一定有用，但可以参考。

### 第一种方法：局域网连接代理
在电脑上打开Clash Verge，连接代理后启用局域网连接 (allow-lan: true)，并且确认局域网连接的端口是多少（端口可在首页的Clash信息里混合端口查看，本次就以7890端口为例子）。启用后开启WiFi热点，手机连接WiFi热点后在连接WiFi的高级设置里选择代理，IP地址填写电脑的IP（IP地址可在设置 - 网络和Internet - 高级网络设置 - 本地连接* 2（名字不一定是这个，网卡名字可能是Microsoft Wi-Fi Direct Virtual Adapter） - 查看其他属性 - IPv4地址。一般来说可能是以下网段，实际情况要看自己的IP具体是多少。

```bash
192.168.137.1
```

知道自己的IP后在手机连接WiFi位置找到进阶选项 - Proxy，Proxy选择手动，主机名称就是电脑的IP地址，端口填写7890，IP设置就默认DHCP。连接成功后进行下一步应该正常使用Google服务进行下一步的操作了

### 第二种方法：在TWRP里输入命令来强行跳过连接网络来进行开机向导
在TWRP里找到终端，在终端里输入：

``` bash
dd if=/dev/zero of=/dev/block/bootdevice/by-name/frp
```

回车后继续回车，等待一段时间后重启进入系统选择“在离线模式下完成设置”，这一步的目的就是清除FRP分区，绕过Google账户锁。

### 第三种方法（最狠的方法）：路由器实现全局模式下局域网连接
给路由器刷入OpenWRT配置一些奇奇怪怪的东西后就可以全局科学上网。你都搞定这个了那还说啥了，给了呗~

如果你有备用机并且它刚刚好有一个功能就是开启热点后共享手机的VPN网络，那你是真爽到了。
